# 药物研发辅助平台整改方案

## 1. 整改前提

本次整改严格以 `开题报告.txt`、`论文.txt` 和 `docs/platform-functional-definition.md` 为边界，不改变课题方向，不扩展到分子设计、临床诊断或单一药物专用平台。

平台定位保持为：

**基于大语言模型的药物研发辅助平台设计与实现。**

本次整改只强化以下能力：

1. 文献导入与自动解析  
2. 单篇文献摘要与研究概览  
3. 结构化指标抽取与人工复核  
4. 基于当前文献范围的 RAG 检索  
5. 文献知识图谱与证据定位  
6. 基于证据的研发辅助分析报告  

---

## 2. 当前问题归纳

结合现有系统测试结果，当前问题主要分为两类：

### 2.1 业务价值不清

1. 导入文献后没有第一时间给出“这篇文献做了什么”的结果  
2. 页面强调分块、向量化、工作流等技术动作，弱化了研发辅助价值  
3. 用户难以判断指标抽取、图谱、报告和证据定位之间的关系  

### 2.2 结果正确性不足

1. 多篇文献进入向量库后，单篇文献分析存在串文献风险  
2. 图谱查询和 RAG 检索没有全面围绕当前文献做范围控制  
3. Milvus 状态查询过于频繁，影响稳定性和使用体验  

---

## 3. 总体整改目标

本次整改后，系统需要满足以下主流程：

**导入文献 → 自动解析 → 自动生成文献概览 → 自动抽取关键指标 → 自动生成当前文献图谱 → 用户复核 → 生成研发辅助报告**

用户在导入一篇文献后，至少应马上看到：

1. 一句话概览  
2. 研究目标和方法摘要  
3. 关键结果和安全性提示  
4. 关键指标列表  
5. 证据片段  
6. 当前文献知识图谱  

---

## 4. 分批整改计划

### 第一批：主流程与入口整改

目标：让平台入口回到“文献先行、结果优先”。

执行内容：

1. 根路由改为文献页，不再默认进入工作流页  
2. 登录和注册成功后跳转到文献页  
3. 侧边栏把文献管理作为主入口  
4. 工作流页保留为“流程演示”能力，不再作为默认首页  

验收标准：

1. 用户登录后直接进入文献管理页  
2. 用户能在主导航中明确理解各页面职责  

### 第二批：单篇文献概览整改

目标：导入文献后立刻给出可读结果。

执行内容：

1. 新增文献概览接口前端接线  
2. 文献页显示处理阶段、摘要、研究目标、方法、结果、安全性和结论  
3. 文献页展示关键概念、关键指标、证据片段和图谱预览  
4. 技术操作下沉到“高级处理控制”区域  

验收标准：

1. 选中文献后可直接看到完整概览  
2. 文献页不再只展示技术按钮  

### 第三批：文献范围隔离整改

目标：保证单篇文献分析只使用当前文献证据。

执行内容：

1. RAG 查询默认带当前 `literatureId`  
2. 图谱查询默认带当前 `literatureId`  
3. 报告生成继续保留文献范围参数  
4. 检索和图谱查询结果显示明确来源

验收标准：

1. 单篇文献检索结果不混入其他文献  
2. 单篇文献图谱只展示当前文献相关节点和关系  

### 第四批：图谱与证据联动整改

目标：让图谱真正服务于阅读和解释。

执行内容：

1. 扩展图谱节点类型，补充疾病、来源、关键词、时间窗口、队列等节点  
2. 图谱查询保留上下文节点，避免只剩孤立节点  
3. 文献概览中提供图谱预览和证据入口  

验收标准：

1. 图谱内容与指标、摘要保持一致  
2. 用户能通过图谱理解“文献研究对象—指标—时间窗口”的关系  

### 第五批：稳定性与性能整改

目标：减少误操作和重复请求带来的异常。

执行内容：

1. 文献页、抽取页、分析页继续保留处理中提示  
2. 缓存 Milvus 健康状态，减少高频重复探测  
3. 保持导入后自动准备链路稳定执行  

验收标准：

1. 连续点击不会造成明显重复执行误判  
2. Milvus 状态查询不再频繁触发连接抖动  

### 第六批：文档与交付整改

目标：让论文、开题报告和系统实现保持一致。

执行内容：

1. 增补整改方案文档  
2. 保持功能命名与“药物研发辅助平台”一致  
3. 对照论文和开题报告保留文献解析、RAG、本地知识库、知识图谱、分析报告等核心实现  

验收标准：

1. 系统功能与论文叙述一致  
2. 答辩时可按“导入文献—看概览—看图谱—看指标—出报告”顺序演示  

---

## 5. 实施顺序

本轮执行顺序如下：

1. 先改入口与导航  
2. 再补文献概览链路  
3. 再收紧 RAG 与图谱范围  
4. 再补图谱内容和页面布局  
5. 最后补稳定性与交付文档  

---

## 6. 验收清单

整改完成后，应至少满足以下检查项：

1. 登录后默认进入文献管理页  
2. 导入文献后自动完成解析、向量化和抽取准备  
3. 文献页能显示当前文献概览  
4. 文献页能显示当前文献图谱预览  
5. 抽取页 RAG 检索默认限制在当前文献  
6. 图谱页支持按当前文献查询  
7. 报告页生成结果可追溯到证据片段  
8. 平台整体叙述不偏离“药物研发辅助平台”题目  

---

## 7. 一句话结论

本次整改不是继续堆技术功能，而是把系统收敛成一个真正可演示、可理解、可追溯的药物研发文献辅助平台。
