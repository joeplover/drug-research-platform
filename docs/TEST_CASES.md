# 测试用例文档

## 1. 文档说明

本文档用于毕业设计项目 `drug-research-platform` 的功能测试与回归测试整理。  
测试内容以当前已经完成的前后端功能为准，不包含尚未接入或尚未实现的扩展能力。

## 2. 测试范围

- 用户注册与登录
- 仪表盘数据展示与组件健康状态展示
- 文献导入、分块、向量同步状态查看
- RAG 检索与分析报告生成
- 指标抽取、指标复核、仅确认指标分析
- 知识图谱查询与复核状态筛选
- 工作流串联处理
- 健康检查与降级状态提示

## 3. 测试环境

| 项目 | 配置 |
|---|---|
| 操作系统 | Windows |
| 后端框架 | Spring Boot 3 |
| 前端框架 | Vue 3 |
| 数据库 | MySQL |
| 缓存 | Redis |
| 向量库 | Milvus |
| LLM 接口 | DeepSeek API |
| Embedding | Ollama `qwen3-embedding:4b` |

## 4. 功能测试用例

| 用例编号 | 模块 | 测试目标 | 前置条件 | 测试步骤 | 预期结果 |
|---|---|---|---|---|---|
| TC-USER-01 | 用户注册 | 验证新用户可正常注册 | 系统启动成功 | 打开注册页，输入用户名、邮箱、密码并提交 | 注册成功，页面给出成功反馈 |
| TC-USER-02 | 用户登录 | 验证已注册用户可登录 | 已存在有效账号 | 输入正确邮箱和密码登录 | 登录成功，进入系统首页 |
| TC-USER-03 | 用户登录 | 验证错误密码处理 | 已存在有效账号 | 输入错误密码登录 | 登录失败，页面提示账号或密码错误 |
| TC-DASH-01 | 仪表盘 | 验证统计卡片展示 | 系统存在基础数据 | 进入仪表盘页面 | 显示文献数、任务数、待复核指标数等统计信息 |
| TC-DASH-02 | 仪表盘 | 验证组件健康状态展示 | MySQL、Redis、Milvus、Embedding 已配置 | 查看系统组件状态区域 | 页面显示各组件状态，不再固定为全部正常 |
| TC-LIT-01 | 文献导入 | 验证按路径导入文献 | 本地存在可解析文献 | 在文献页面提交导入请求 | 文献记录创建成功，能看到标题和基础元数据 |
| TC-LIT-02 | 文献分块 | 验证文献切片处理 | 已导入文献 | 执行文献处理或工作流 | 系统生成分块数据，文献详情可查看分块内容 |
| TC-LIT-03 | 向量同步 | 验证文献向量同步状态记录 | Milvus 已可用 | 对文献执行向量化 | 文献显示同步状态、同步块数和同步时间 |
| TC-LIT-04 | 向量状态查看 | 验证单篇文献向量状态接口 | 已存在文献 | 打开文献向量状态区域 | 页面可看到是否已同步、块数、详情说明 |
| TC-RAG-01 | RAG 分析 | 验证检索增强报告生成 | 文献已导入并完成分块 | 在分析页输入问题并提交 | 返回报告摘要、引用证据和检索结果 |
| TC-RAG-02 | 证据定位 | 验证引用证据可回跳到文献块 | 已生成分析结果 | 点击分析结果中的证据定位 | 页面跳转到对应文献块并高亮 |
| TC-RAG-03 | 检索模式显示 | 验证检索模式展示 | 已生成分析结果 | 查看分析结果证据区域 | 每条证据显示定位信息和检索模式 |
| TC-EXT-01 | 指标抽取 | 验证文献指标抽取 | 已导入文献 | 在抽取页执行抽取 | 系统生成结构化指标记录 |
| TC-EXT-02 | 指标去重 | 验证重复指标控制 | 文献中存在重复表达 | 执行抽取后查看结果 | 重复指标数量明显减少，结果更聚合 |
| TC-EXT-03 | 指标复核-确认 | 验证指标确认流程 | 已有待复核指标 | 点击确认并填写说明 | 指标状态变为 `已确认`，记录复核时间 |
| TC-EXT-04 | 指标复核-驳回 | 验证指标驳回流程 | 已有待复核指标 | 点击驳回并填写说明 | 指标状态变为 `已拒绝`，记录复核说明 |
| TC-ANA-01 | 仅确认指标报告 | 验证分析只使用确认指标 | 存在不同复核状态指标 | 勾选“仅确认指标”后生成报告 | 报告使用 `已确认` 指标，不混入待复核或驳回指标 |
| TC-GRAPH-01 | 图谱查询 | 验证图谱节点展示 | 已有抽取指标 | 打开图谱页面并查询 | 页面显示文献、指标等节点与关系 |
| TC-GRAPH-02 | 图谱筛选 | 验证按复核状态过滤 | 已有不同状态指标 | 在图谱页选择确认/待复核/驳回 | 图谱节点随筛选条件变化 |
| TC-WF-01 | 全流程工作流 | 验证导入到报告的串联处理 | 已有可处理文献 | 执行工作流处理 | 系统完成导入、抽取、分析等步骤并生成任务结果 |
| TC-HEALTH-01 | 健康检查 | 验证真实健康状态返回 | 后端服务已启动 | 访问健康检查接口 | 返回 MySQL、Redis、Milvus、Embedding 的真实状态 |

## 5. 异常场景测试

| 用例编号 | 模块 | 异常场景 | 测试步骤 | 预期结果 |
|---|---|---|---|---|
| TC-ERR-01 | 登录 | 账号不存在 | 输入不存在的邮箱登录 | 返回失败提示，不进入系统 |
| TC-ERR-02 | 文献导入 | 文件路径错误 | 提交不存在的文献路径 | 导入失败并提示路径无效 |
| TC-ERR-03 | Milvus | 向量库不可用 | Milvus 停止后执行向量化 | 系统给出异常或降级状态，不应伪装为成功 |
| TC-ERR-04 | Embedding | Ollama 不可用 | 关闭 Ollama 后执行抽取或向量化 | 系统记录降级模式，并在健康状态中可见 |
| TC-ERR-05 | 报告生成 | 未选择文献直接分析 | 提交空文献列表 | 系统返回参数错误或提示缺少文献 |
| TC-ERR-06 | 证据定位 | 分析引用缺失块信息 | 打开异常证据链接 | 页面不崩溃，并给出兼容处理 |
| TC-ERR-07 | 指标复核 | 复核不存在的指标 | 调用复核接口时传错 ID | 返回失败信息，不更新其他数据 |
| TC-ERR-08 | 重复向量化 | 同一文献重复同步 | 多次执行向量同步 | 系统状态保持一致，不生成重复主键冲突数据 |

## 6. 回归重点

- 回归用户注册、登录、页面跳转链路
- 回归文献导入、分块、向量同步状态写回
- 回归分析报告中的证据定位、引用展示、检索模式展示
- 回归指标抽取后的复核状态流转
- 回归“仅确认指标”对报告与工作流的影响
- 回归图谱筛选与仪表盘统计的一致性
- 回归健康检查接口与前端组件状态展示的一致性

## 7. 验收前建议重点检查

- 检查 `application.yml` 中数据库、Redis、Milvus、DeepSeek、Ollama 配置是否正确
- 检查 Milvus 与 Ollama 不可用时，页面是否仍能给出清晰状态提示
- 检查演示账号是否具备完整样例数据
- 检查报告、图谱、抽取页之间的跳转是否通畅
